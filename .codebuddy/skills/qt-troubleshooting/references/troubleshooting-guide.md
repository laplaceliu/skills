# Qt 问题诊断流程指南

## 诊断流程概览

```
┌─────────────┐
│  问题报告    │ ← 用户提交问题
└──────┬──────┘
       ▼
┌─────────────┐
│  信息收集    │ ← 交互式问询（现象、环境、复现步骤）
└──────┬──────┘
       ▼
┌─────────────┐
│  问题分类    │ ← 崩溃/UI/性能/通信/运行时
└──────┬──────┘
       ▼
┌─────────────┐
│  根因分析    │ ← 调用技术知识 + 诊断工具
└──────┬──────┘
       ▼
┌─────────────┐
│  方案验证    │ ← 代码修改 + 测试验证
└──────┬──────┘
       ▼
┌─────────────┐
│  报告归档    │ ← markdown-doc 生成报告
└─────────────┘
```

## 信息收集检查清单

### 必填信息

| 信息类型 | 收集内容 | 重要性 |
|----------|----------|--------|
| 问题现象 | 具体表现、错误信息、截图 | 必填 |
| 环境信息 | OS、Qt版本、编译器、构建类型 | 必填 |
| 复现步骤 | 完整操作序列 | 必填 |
| 错误日志 | 终端输出、堆栈跟踪 | 强烈推荐 |

### 选填信息

| 信息类型 | 收集内容 | 重要性 |
|----------|----------|--------|
| 期望行为 | 正常情况应该怎样 | 推荐 |
| 已尝试方案 | 尝试过的方法及结果 | 推荐 |
| 相关代码 | 问题相关代码段 | 推荐 |
| 时序信息 | 问题发生的时间点 | 可选 |

## 问题分类决策表

### 按严重性分类

| 等级 | 描述 | 响应优先级 | 示例 |
|------|------|-----------|------|
| P0 | 应用崩溃/数据损坏 | 立即处理 | 段错误、断言失败、数据丢失 |
| P1 | 核心功能失效 | 24小时内 | 登录失败、保存失败、显示黑屏 |
| P2 | UI/体验问题 | 计划修复 | 样式异常、响应延迟、布局错乱 |
| P3 | 边缘case | 后续迭代 | 特定输入法、特殊数据格式 |

### 按问题类型分类

#### 1. 崩溃类问题

**典型特征**：
- 进程异常退出
- 段错误（segfault）
- 断言失败（assertion failed）
- 未捕获异常

**诊断路径**：
```
崩溃报告
    │
    ├── 堆栈信息完整？
    │       ├── YES → 分析堆栈定位崩溃点
    │       └── NO  → 启用 ASAN 重新复现
    │
    ├── Qt 警告信息？
    │       ├── YES → 分析 Qt 警告（通常是根因提示）
    │       └── NO  → 检查对象生命周期
    │
    └── 特定操作触发？
            ├── YES → 最小化复现场景
            └── NO  → 内存问题（ASAN/Valgrind）
```

**关键诊断命令**：
```bash
# Debug 构建 + ASAN
cmake -B build -DCMAKE_BUILD_TYPE=Debug \
    -DCMAKE_CXX_FLAGS="-fsanitize=address -fsanitize=leak"
./build/myapp

# GDB 堆栈
gdb ./build/myapp
run [args]
bt full
```

#### 2. UI 类问题

**典型特征**：
- Widget 不显示
- 显示内容异常
- 样式不生效
- 布局错乱

**诊断路径**：
```
UI 异常
    │
    ├── Widget 层级正确？
    │       ├── 检查 parent/child 关系
    │       └── dumpWidgetTree() 诊断
    │
    ├── 可见性状态？
    │       ├── isVisible() == true?
    │       ├── size().isValid()?
    │       └── setVisible(true)?
    │
    ├── 样式生效？
    │       ├── styleSheet() 内容？
    │       └── 尝试 setStyleSheet("background: lime;")
    │
    └── 事件循环正常？
            └── processEvents() 响应？
```

#### 3. 信号槽问题

**典型特征**：
- 信号触发后槽未调用
- 连接失败（运行时警告）
- 数据传递异常

**诊断路径**：
```
信号槽问题
    │
    ├── 连接成功？
    │       ├── 检查连接返回值（bool）
    │       └── 查看 "QObject::connect: ..." 警告
    │
    ├── Q_OBJECT 存在？
    │       ├── moc 文件生成？
    │       └── metaObject() 有效？
    │
    ├── 签名匹配？
    │       ├── 信号：void signal(int)
    │       └── 槽：void slot(int) — 参数类型必须一致
    │
    └── 线程边界？
            ├── Qt::QueuedConnection？
            └── 跨线程需要事件队列
```

**调试连接代码**：
```cpp
// 添加调试连接
connect(sender, &Sender::signal, [](auto&&... args) {
    qDebug() << "SIGNAL FIRED:" << args...;
});

// 检查对象是否存活
QPointer<QObject> p = sender;
if (p) {
    // sender 仍然存活
}
```

#### 4. 内存问题

**典型特征**：
- 内存持续增长
- 进程内存占用过高
- 特定操作后内存不释放

**诊断路径**：
```
内存问题
    │
    ├── 泄漏检测？
    │       ├── ASAN 配置
    │       └── Valgrind --leak-check=full
    │
    ├── 对象未释放？
    │       ├── QObject::dumpObjectTree()
    │       └── QObject::staticMetaObject.className()
    │
    └── 循环引用？
            ├── QSharedPointer 循环？
            └── QPointer 断环
```

## 根因分析技术

### 5Why 分析法

```
问题：点击按钮后应用崩溃

Why 1: 为什么崩溃？
    → 按钮点击处理函数中访问了空指针

Why 2: 为什么是空指针？
    → 对话框关闭时被 delete，指针未置空

Why 3: 为什么 delete 后未置空？
    → 对话框使用 raw pointer，closeEvent 中直接 delete

Why 4: 为什么 closeEvent 直接 delete？
    → 未使用 Qt 的 deleteLater() 机制

Why 5: 为什么不用 deleteLater()？
    → 开发者不熟悉 Qt 内存管理模式

最终根因：开发者对 Qt 对象生命周期管理不熟悉
预防措施：代码审查 + Qt 内存管理规范
```

### 假设验证法

```
假设：问题是内存泄漏

验证步骤：
1. 记录初始内存 → 操作后内存
2. 如果增长 → 假设成立
3. 如果不增长 → 假设不成立，重新分类

假设：问题是线程竞争

验证步骤：
1. 单线程模式运行（禁用多线程）
2. 如果问题消失 → 假设成立
3. 如果问题仍在 → 假设不成立
```

## 诊断工具清单

### 静态分析

| 工具 | 用途 | 命令 |
|------|------|------|
| clazy | Qt 特定警告 | `cmake -DCLANG_TIDY=clazy-...` |
| cppcheck | 通用静态分析 | `cppcheck --enable=all src/` |

### 动态分析

| 工具 | 用途 | 命令 |
|------|------|------|
| ASAN | 内存错误检测 | `CMAKE_CXX_FLAGS="-fsanitize=address"` |
| Valgrind | 内存泄漏检测 | `valgrind --leak-check=full ./app` |
| perf | 性能分析 | `perf record -g ./app` |
| strace | 系统调用追踪 | `strace -f ./app 2>&1 | tail` |

### Qt 专用工具

| 工具 | 用途 | 命令 |
|------|------|------|
| Qt Creator Debugger | 可视化调试 | 内置 |
| QDjango Inspector | ORM 调试 | `QDjango::Debug::enabled = true` |
| QLoggingCategory | 分类日志 | `QT_LOGGING_RULES="*.debug=true"` |
