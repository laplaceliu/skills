# 系统设计模式参考

> **技术聚焦说明**: 本文档中的架构模式和设计模式默认基于 **C++/Qt** 技术栈。如无特殊原因，优先选择与 Qt 框架契合的方案。

- **默认语言**: C++ (C++11)
- **默认 GUI**: Qt Widgets 或 Qt Quick/QML
- **默认 HTTP**: Qt HTTP Server (QHttpServer)
- **默认构建**: qmake + CMake (混合使用)

如需其他技术栈方案，请明确说明。

## 1. 架构模式

### 1.1 分层架构 (Layered Architecture)

**适用场景**: 企业应用、CRUD 系统、传统 Web 应用

```
┌─────────────────────────────────────────────────────────────────────┐
│                         分层架构                                    │
├─────────────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                    表示层 (Presentation Layer)               │   │
│  │     Controller, View, DTO, API Response                      │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                              │                                     │
│                              ▼                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                    应用层 (Application Layer)                │   │
│  │     Service, Use Case, Command, Query, Orchestration         │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                              │                                     │
│                              ▼                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                    领域层 (Domain Layer)                    │   │
│  │     Entity, Value Object, Domain Service, Repository Interface│  │
│  └─────────────────────────────────────────────────────────────┘   │
│                              │                                     │
│                              ▼                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                   基础设施层 (Infrastructure Layer)          │   │
│  │     Repository Impl, External Service, Database, Cache       │   │
│  └─────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────┘
```

**优点**: 清晰分层、职责分明、易于理解和维护
**缺点**: 容易产生贫血模型、层次过多可能影响性能

### 1.2 事件驱动架构 (Event-Driven Architecture)

**适用场景**: 实时系统、微服务集成、异步处理

```
┌─────────────────────────────────────────────────────────────────────┐
│                       事件驱动架构                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────┐      ┌─────────────┐      ┌─────────┐               │
│   │ Event   │ ───▶ │   Event    │ ───▶ │ Event   │               │
│   │ Publisher│      │   Broker   │      │ Handler │               │
│   └─────────┘      │  (Kafka/   │      └─────────┘               │
│                    │  RabbitMQ) │              │                   │
│                    └─────────────┘              ▼                   │
│                                           ┌─────────┐               │
│   ┌─────────┐      ┌─────────────┐        │ Event   │               │
│   │ Command │ ───▶ │   Command  │ ──────▶│ Handler │               │
│   │         │      │   Handler  │        └─────────┘               │
│   └─────────┘      └─────────────┘                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**事件类型**:
| 类型 | 说明 | 示例 |
|------|------|------|
| 域事件 (Domain Event) | 业务状态变更 | OrderCreated, PaymentCompleted |
| 集成事件 (Integration Event) | 跨系统同步 | UserRegistered, ProductUpdated |
| 事件 (Event) | 已发生的事实 | EventName (过去式) |

### 1.3 微服务架构 (Microservices Architecture)

**适用场景**: 大型复杂系统、需要独立部署和扩展的业务

```
┌─────────────────────────────────────────────────────────────────────┐
│                       微服务架构                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│   │ 用户服务  │  │ 订单服务  │  │ 支付服务  │  │ 通知服务  │          │
│   │ :8001   │  │ :8002   │  │ :8003   │  │ :8004   │          │
│   └───┬──────┘  └───┬──────┘  └────┬────┘  └────┬────┘          │
│       │              │              │              │                │
│       └──────────────┴──────────────┴──────────────┘                │
│                              │                                      │
│                    ┌─────────┴─────────┐                            │
│                    │   API Gateway    │                            │
│                    │   /api/v1/*      │                            │
│                    └───────────────────┘                            │
│                                                                     │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐                        │
│   │   Postgres │  │   Redis   │  │   Kafka   │                        │
│   │   用户数据  │  │   缓存    │  │   消息    │                        │
│   └──────────┘  └──────────┘  └──────────┘                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**服务拆分原则**:
- 单一职责
- 独立部署
- 按业务能力拆分
- 按领域驱动设计划分边界

## 2. 领域驱动设计模式

### 2.1 聚合 (Aggregate)

```
┌─────────────────────────────────────────────────────────────────────┐
│                           聚合模式                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌───────────────────────────────────────────────────────────┐     │
│   │                     Order (聚合根)                        │     │
│   │                                                           │     │
│   │   ┌─────────────────────────────────────────────────┐    │     │
│   │   │              OrderLineItem (实体)                │    │     │
│   │   │   - id: OrderLineItemId                         │    │     │
│   │   │   - productId: ProductId                        │    │     │
│   │   │   - quantity: int                               │    │     │
│   │   │   - price: Money                                │    │     │
│   │   └─────────────────────────────────────────────────┘    │     │
│   │                                                           │     │
│   │   ┌─────────────────────────────────────────────────┐    │     │
│   │   │           ShippingAddress (值对象)                │    │     │
│   │   │   - street: String                              │    │     │
│   │   │   - city: String                                │    │     │
│   │   │   - zipCode: String                             │    │     │
│   │   └─────────────────────────────────────────────────┘    │     │
│   │                                                           │     │
│   │   + addItem(product, quantity)                           │     │
│   │   + removeItem(itemId)                                  │     │
│   │   + setShippingAddress(address)                          │     │
│   │   + calculateTotal(): Money                             │     │
│   │   + confirm()                                           │     │
│   └───────────────────────────────────────────────────────────┘     │
│                                                                     │
│   规则:                                                            │
│   - 外部对象只能通过聚合根访问聚合内的对象                           │
│   - 聚合根负责维护聚合内部的不变性                                   │
│   - 事务边界在聚合根                                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.2 命令处理器 (Command Handler)

```
┌─────────────────────────────────────────────────────────────────────┐
│                       命令处理器模式                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   // 命令 (不可变)                                                   │
│   public record CreateOrderCommand(                                 │
│       CustomerId customerId,                                        │
│       List<OrderItem> items                                         │
│   ) {}                                                              │
│                                                                     │
│   // 命令处理器                                                      │
│   @Component                                                        │
│   public class CreateOrderCommandHandler                            │
│           implements CommandHandler<CreateOrderCommand, OrderId> {  │
│                                                                     │
│       private final OrderRepository repository;                     │
│                                                                     │
│       @Override                                                     │
│       public OrderId handle(CreateOrderCommand command) {           │
│           Customer customer = customerRepository                    │
│               .findById(command.customerId());                      │
│                                                                     │
│           Order order = Order.create(customer);                     │
│           command.items().forEach(order::addItem);                 │
│           order.confirm();                                          │
│                                                                     │
│           repository.save(order);                                   │
│           return order.getId();                                     │
│       }                                                            │
│   }                                                                │
│                                                                     │
│   // 命令调度器                                                      │
│   @Service                                                         │
│   public class CommandDispatcher {                                  │
│       public <T, R> R dispatch(Command<T, R> command) {             │
│           // 查找对应的 Handler 并执行                              │
│       }                                                            │
│   }                                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.3 查询处理器 (Query Handler / CQRS Read)

```
┌─────────────────────────────────────────────────────────────────────┐
│                       查询处理器模式                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   // 查询 (不可变)                                                   │
│   public record OrderQuery(                                        │
│       OrderStatus status,                                           │
│       LocalDate fromDate,                                           │
│       LocalDate toDate                                              │
│   ) {}                                                              │
│                                                                     │
│   // 查询处理器                                                      │
│   @Component                                                        │
│   public class OrderQueryHandler                                   │
│           implements QueryHandler<OrderQuery, List<OrderDto>> {     │
│                                                                     │
│       private final JdbcTemplate jdbcTemplate;                      │
│                                                                     │
│       @Override                                                     │
│       public List<OrderDto> handle(OrderQuery query) {              │
│           // 直接使用 JDBC/QueryDSL 查询                            │
│           return jdbcTemplate.query(                               │
│               "SELECT * FROM orders WHERE status = ?",              │
│               rowMapper,                                            │
│               query.status().name()                                 │
│           );                                                        │
│       }                                                            │
│   }                                                                │
│                                                                     │
│   // 与 CQRS 结合使用                                                │
│   // Commands → Command Handler → 写模型                            │
│   // Queries → Query Handler → 读模型                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 5. 架构决策记录 (ADR) 模板

```
┌─────────────────────────────────────────────────────────────────────┐
│                    架构决策记录 (ADR)                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│ 编号: ADR-001                                                      │
│ 标题: 采用微服务架构拆分订单系统                                     │
│ 日期: 2024-01-15                                                    │
│ 状态: 已接受                                                        │
│                                                                     │
│ ## 背景                                                            │
│ 当前的单体应用在订单量增长时难以独立扩展，特别是支付模块和库存模块。   │
│ 需要将系统拆分为可独立部署的服务。                                   │
│                                                                     │
│ ## 决策                                                            │
│ 将订单系统拆分为以下微服务:                                          │
│ - 用户服务 (用户管理、认证授权)                                       │
```
